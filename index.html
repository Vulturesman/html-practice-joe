What is the difference between HTML and CSS?


In the broad classisifications of coding / web development, (front-end and back-end). HTML and CSS are both concerned with the front end or the 'client side'. They both deal with the information provided by a web page, but they are not the same thing:

HTML

HTML deals with the actual content of the page, and 'marks up' or displays precisely what the developer, or thier client needs to communicate. In my case, that is the nuances of html and CSS.

This is acheived by grouping different snippets of information together into things called elements. Simply defined as some information or 'content' enclosed between specific tags. The tags are there to help the client to display that information correctly. The image below provides and example of what this looks like from the actual code for this website!

CSS

So, what exactly is left for CSS to do? Well, the way that information is displayed is often atleast as important as the actual content. And that is where CSS comes in. CSS stands for Cascading Syle Sheets. As the name suggests, it is the styling of a webpage that is the wheelhouse of CSS. Anything from pictures to different font colours, and sizes, different background colours and much more!

What are the top 5 most used HTML elements?

Many HTML elements will be used over and over again, the five most used are listed below;

  • <p> a paragraph;
  • <div> a division or section of your code;
  • <a> a hyperlink to another webpage;
  • <img> add an image to your page;
  • <li> used for every list item, like this one!

What is the difference between semantic and non-semantic elements?

A semantic tag is essentially a tag that contains some meaning, it is useful for us as humans since we can immeditaely get an idea of the content of that element, even without any previous knowledge of coding! See if you can infer meaning from some examples below:

  • <header>
  • <footer>
  • <table>
  • <form>

A non-sematic element is precisely the opposite, as in, the tag gives us no indiction of the content, examples are <div> and <span>.
